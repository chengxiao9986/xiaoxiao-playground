# Bot 团队协作指南

## 团队成员

| Bot | 角色 | 找谁 |
|-----|------|------|
| @饼饼 | 大总管 | 有新需求、想做什么事、想了解进度 |
| @小圆 | 程序员 | 直接聊技术细节、看代码、调 bug |
| @鸽鸽 | 产品 + UX + Marketing | 聊产品想法、讨论用户体验、做营销内容 |
| @薯条 | 生活管家 | 育儿记录、健康管理、生活规划、日常聊天 |

---

## 如何使用

### 场景 1：我想做一个 App（全流程）

在 `#general` 频道跟饼饼说你的想法：

```
@饼饼 我想做一个记账 App，可以自动分类消费记录
```

饼饼会按阶段推进整个流程：

**产品阶段** → 饼饼 dispatch 给鸽鸽
- 鸽鸽写 PRD、定义功能范围、设计用户流程

**开发阶段** → 饼饼 dispatch 给小圆
- 小圆根据 PRD 写代码、搭架构、部署

**上线营销阶段** → 饼饼 dispatch 给鸽鸽
- 鸽鸽撰写 App Store / Google Play 上架描述
- 鸽鸽制定营销策略（目标用户、渠道、增长方案）
- 鸽鸽创作推广内容（小红书文案、社交媒体帖子、宣传语）
- 鸽鸽策划上线活动和用户获取方案

你也可以随时追问进度：`@饼饼 现在进度怎么样了？`

### 场景 2：直接找某个 bot

不想走大总管流程时，直接 @ 对应 bot：

```
@小圆 帮我看看这段代码有没有 bug
@鸽鸽 帮我写一个 App Store 上架描述
@鸽鸽 帮我写几条小红书推广文案
@薯条 宝宝今天吃了 200ml 奶，2 点午睡到 3:30
```

### 场景 3：薯条家庭使用

**个人事务** → 直接 DM 私聊薯条（每人记忆独立）

```
DM @薯条: 帮我记一下今天跑了 5 公里
```

**家庭共享** → 在 `#baby-log` 频道记录（全家可见）

```
@薯条 宝宝今天第一次翻身了！
```

## 常见问题

### Q: 饼饼怎么知道要把任务分给谁？
饼饼的 system prompt 里定义了分工规则：开发类给小圆，产品/设计/营销类给鸽鸽。

### Q: 我能同时跟多个 bot 聊吗？
可以，在不同频道或私聊里同时跟不同 bot 对话，互不影响。

### Q: 薯条会把我和家人的聊天搞混吗？
不会。DM 私聊是独立会话，每个人的记忆分开。只有在公共频道（如 #baby-log）里的内容是共享的。

### Q: bot 之间能互相聊天吗？
可以。已启用 agent-to-agent 通信，饼饼可以直接给小圆/鸽鸽发消息分配任务。

### Q: 鸽鸽什么时候做 Marketing？
鸽鸽全程参与：产品阶段做需求和设计，App 上线后切换到营销模式，负责推广内容和增长策略。不需要额外的 bot。
